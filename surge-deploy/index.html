🔍 欠席自動判定ロジック（FormHandler の核心）
📋 フォーム1行を処理する流れ
① 実施日を取得 フォーム回答の「実施日」列を読み込む
日付型に変換
→
② 対象曜日を判定 実施日から曜日を自動算出
（例: 2026/04/17 → 金曜日）
月〜土に対応
→
③ 対象生徒を絞り込み マスターシートから
「その曜日がレギュラー」な
生徒一覧を取得
→
④ 各生徒の出欠を判定 フォームの生徒列を確認

出席 → PRESENT
空白 → ABSENT（欠席）
振替 → MAKEUP
→
⑤ 出欠ログへ書き込み 生徒ごとに1行追記
・logID（UUID）
・フォーム受信時刻（JST）
・振替フラグ
⚠️ 重要ルール： レギュラー曜日でない生徒は空欄でも欠席にならない。 レギュラー曜日の生徒だけが「空欄 = 欠席」と自動判定される。
📊 スプレッドシート構成（全シート）
📋
フォーム回答シート
タイムスタンプ 実施日 生徒A 生徒B …
Google Forms が自動作成・自動追記。手動で編集しない。
📜
出欠ログシート（出欠ログ）
logId studentId 氏名 実施日 所属曜日 出欠 欠席振替発生 振替消化 元フォーム行 フォーム受信時刻 メモ
GASが自動追記する。すべての出欠履歴が蓄積される。
📈
集計シート
studentId 氏名 出席数 欠席数 振替発生数 振替消化数 残り振替
出欠ログから自動計算。毎朝9時と処理後に自動更新。
👥
マスターシート
studentId 氏名 レギュラー曜日 コース 月回数 token
手動で管理。曜日・コース変更時はここを編集する。
🔗
URLリストシート
氏名 token 元URL（長い） 短縮URL
「URLリストを出力」メニューで生成。保護者に短縮URLを共有する。
⚙️
設定シート
タイムゾーン WebアプリURL フォーム_実施日列名 フォーム_所属曜日列名
システムの動作設定を管理。WebアプリURLを再デプロイ後に更新する。
📁 GAS スクリプトファイル構成
🔄
FormHandler.js
  • onFormSubmit：フォーム送信時処理
  • processFormRow_：1行を出欠ログへ変換
  • reprocessAllFormRows_：全行を一括再処理
  • reprocessLastFormRow_：最終行だけ再処理
📊
Aggregation.js
  • recomputeSummary_：集計シートを再計算
  • writeSummaryRows_：集計結果を書き込み
🔑
Master.js
  • loadMasterRecords_：マスター読み込み
  • exportStudentUrls_：URLリスト生成
  • uuid_：一意IDを生成
🌐
WebApp（スクリプト）.js
  • doGet：URLアクセスを受け付け
  • doGetMain_：トークン検証・データ取得
  • WebApp.html でページを表示
🔔
Alert.js
  • dailyAttendanceAlert_：未入力チェック
  • 通知シートへエラーを記録
🛠️
Set up.js / Config.js / Utils.js
  • onOpen：メニュー追加
  • CONFIG：全定数を一元管理
  • getSetting_：設定シートから値取得
  • getFormAnswersSheet_：シート自動検出
📅 授業日の流れ（タイムライン）
授業前日〜当日の朝
📝 保護者がGoogleフォームを入力・送信
各生徒の「出席 / 欠席 / 振替消化」を入力。送信した瞬間に onFormSubmit トリガーが発火。
フォーム送信から数秒以内
🔄 FormHandler.js が自動実行
実施日の曜日を判定 → その曜日がレギュラーな生徒を絞り込み → 欠席自動判定（空欄=欠席）→ 出欠ログへ追記 → 集計シートを即時更新。
毎朝 9:00（自動バッチ）
📊 Aggregation.js が集計を再計算
出欠ログを全件読み直して集計シートを最新化。処理漏れや手動修正後のリカバリーに対応。
保護者が確認したいとき（いつでも）
📱 短縮URLをタップして確認
LINEなどで共有された短縮URLをタップ → ウェブアプリがトークン認証 → 自分の子どもの出欠ページを表示。
生徒情報を変更するとき（随時）
⚙️ マスターシートを直接編集
曜日変更・コース変更 → マスターシートを更新 → 「全行を再処理」でログを再構築 → URLリストを再出力して保護者へ再共有。
📌 運用チートシート（よくある操作）
やりたいこと タイミング 操作方法
毎日の出欠入力 授業ごと 保護者がGoogleフォームを送信するだけ（自動処理）
集計を今すぐ更新 随時 メニュー「育成出欠V1」→「集計を今すぐ再計算」
過去のデータを再取り込み 不具合修正後など メニュー「フォーム回答の全行を再処理（出欠ログ再構築）」
生徒の曜日・コースを変更 学期ごとなど マスターシートを直接編集 → 「全行を再処理」でログを再構築
生徒を追加 入会時 マスターシートに1行追加 → 「URLリストを出力」で新URLを発行
生徒のURLを再発行 URL紛失時など メニュー「生徒ごとのURLをURLリストシートに出力」
WebアプリURLを更新 再デプロイ後 設定シートの「WebアプリURL」セルを新URLに書き換え
時刻が日本時間と合わない タイムゾーン確認 スプレッドシート「ファイル」→「設定」→「(GMT+09:00) Japan Standard Time」に変更
自動トリガーを登録 初期設定時 メニュー「時間トリガーを登録（毎朝9時・集計）」「毎朝7時・アラート」
