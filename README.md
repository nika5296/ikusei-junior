# 育成クラス出欠・振替管理システム V1

Googleフォーム・Googleスプレッドシート・Google Apps Script（GAS）で、コーチの出欠入力・振替残数の自動集計・保護者向け閲覧（トークンURL）・未入力メールアラートをまとめたテンプレートです。

### Surge（画像アセットの公開）

保護者向け **WebApp.html** は **Tailwind CSS（CDN）** 内蔵。Surge には **ロゴ・ベア画像** などを置き、設定の `styles.css` URL から **ベースURL** を算出して画像を読み込みます。

- 静的ファイル: フォルダ **`surge/`**（`kamakura-green-logo.png`、`bear-left.png`、`bear-right.png`、任意の `styles.css`、プレビュー `index.html`）
- デプロイ手順: **`surge/README.md`**
- 公開後、`https://（あなたのサブドメイン）.surge.sh/styles.css` を **設定シートの「Surge_スタイルシートURL」** に記入するか、`Config.gs` の `SURGE_STYLESHEET_URL` を書き換えてから Web アプリを再デプロイしてください。

### 保護者ポータル（Next.js · Vercel）

Next.js アプリは **`parent-portal/`** にあります。このリポジトリはモノレポのため、Vercel で **Root Directory を `parent-portal` にしない**と、ルート URLが **`404: NOT_FOUND`** になることがあります。

**デプロイ:** Vercel で **GitHub リポジトリを接続**し、上記 Root Directory を設定すると、**`main` / `master` への push だけで自動デプロイ**されます（リポジトリ既定。GitHub Actions の push 連動は使いません）。

1. Vercel プロジェクトの **Settings → General → Root Directory** を **`parent-portal`** に変更して保存する  
2. **Environment Variables** に **`GAS_STUDENT_API_BASE_URL`**（GAS Web アプリの `.../exec` までのベース URL）を設定する  
3. 初回または設定変更後は **Deployments** から **Redeploy**  

詳細は **`parent-portal/README.md`** を参照してください。

## 既存スプレッドシート（出欠管理票_2026）について

**本リポジトリからは、共有設定の制限により「4月」表のセル内容を機械的に読み取れない場合があります。**  
次の点のみ外部から確認できている例があります。

- ファイル名は **「Jr育成クラス出欠名簿」** のような名称で保管されている
- タブに **「出欠管理票_2026」「2026」「フォームの回答」** などが存在する

**必ず手元で次を確認してください。**

1. ブラウザで [別例・出欠管理票（共有リンク）](https://docs.google.com/spreadsheets/d/1TVB5E0k2UReOpjA09oOFahQTpT2aQn_-PNlcgblEiAc/edit?usp=sharing) を開く（上記の運用ブックと別ファイルの場合あり）  
2. **「4月」** の表（シートまたはブロック）で、次をメモする  
   - 生徒名がどの列か  
   - 参加曜日・コース（週1/2/3）がどこに書かれているか  
   - 日付ごとの出欠記号がどの列にあるか  

### 現行運用の読み解き（一般的な名簿＋ご要件からの整理）

| 現行（多くの名簿でよくある形） | V1での位置づけ |
|-------------------------------|----------------|
| 縦に生徒名、横に日付のマトリクスで○×など | **廃止**し、**1行＝1生徒×1レッスン日**の「出欠ログ」へ正規化 |
| 「参加曜日」「週回数」列 | **生徒マスタ**の「レギュラー曜日」「コース」に集約 |
| 手書き・直接セル編集 | **Googleフォーム**へ移行（入力漏れは通知で補助） |

### 既存表から引き継ぐ情報

- **氏名**（フォームの設問見出しと**完全一致**させる必要あり）
- **レギュラー曜日**（月・火・木・金の組み合わせ）
- **コース**（週1/週2/週3）
- 移行時点の振替残が分かる場合は **初期残振替** に数値を入れる

### 既存運用から変わる点

- 日々の出欠は **フォーム送信**が正規ルート（シート直編集は例外運用）
- 保護者は **トークン付きURL** で残数のみ閲覧（ログインなし）
- 前日分のフォームが無い場合、**翌朝7時にメール**で通知（休校日は設定で除外可能）

## V1の設計要約

1. **フォーム**で「実施日」「所属曜日（月火木金）」＋**その曜日の生徒ごとの出欠**を1回の送信で記録する（セクション分岐で曜日別に表示推奨）。  
2. **フォーム回答**を GAS が読み取り、**出欠ログ**に1行ずつ展開する。  
3. **集計**で、生徒ごとに  
   - **振替発生**＝レギュラー欠席（マスタ上その曜日がレギュラーで欠席）  
   - **振替消化**＝「振替出席」の回数  
   - **残振替**＝初期残振替＋発生−消化  
4. **Webアプリ**が `token` で生徒を特定し、残数・最終集計時刻・予約ボタンを表示。  
5. **時間主導トリガー**で前日が月・火・木・金のときだけ、フォーム回答の有無を確認し、無ければメール送信。

### このV1でできること

- コーチがフォームから出欠入力し、ログ・集計に反映される  
- 振替の **発生条件** と **消化条件** をルールどおり集計する  
- 保護者がスマホで **残振替** と **予約リンク** を確認できる  
- **未入力の可能性**をメールで知らせる（休校日は除外日設定で抑制）

### このV1ではまだやらないこと

- ログイン・パスワード認証（トークンURLのみ）  
- LINE通知  
- 予約システム本体とのAPI連携  
- トークンを Script Properties のみで管理（シートに平文で保持する簡易方式）  
- 完全な個人情報保護（URLを知る人は閲覧可能—**家族内共有の注意**が前提）

## レビュー反映メモ（UI/UX・エンジニア）

- **UI/UX**: 「実施日」と「所属曜日」の意味を混同しないよう、フォーム文言は **「所属曜日（定例クラス）」** を推奨。保護者画面では **残数を最優先の大きさ**にし、ルールを1行で併記。  
- **エンジニア**: フォーム処理と集計に **LockService** を使用。未入力判定は **タイムゾーン Asia/Tokyo** を明示。将来は集計のみ時間バッチに分離可能な構造（現状は小規模向けにフォーム後に全件再集計）。

## Googleフォーム項目案（シンプル・iPad向け）

**共通（先頭）**

1. **実施日**（日付）  
2. **所属曜日**（プルダウン：月 / 火 / 木 / 金）— *レッスンが「どの定例クラス」か*  
3. **コーチ名**（任意・短文）  

**セクション分岐（推奨）**

- 「所属曜日＝月」のときだけ **月曜クラスの生徒** を一覧表示  
- 各生徒につき **1問（ラジオ）**：`出席` / `欠席` / `振替出席`  
- **設問のタイトル（見出し）を、生徒マスタの「氏名」と一字一句同じにする**（GASが列名で突合）

**運用のコツ**

- 出席が多い場合、ラジオの並びは **出席を先頭**にするとタップが速い  
- 誤送信防止に、最後に **確認** セクションと短い注意文を入れるとよい

## 初期設定手順（概要）

1. **新しい Google スプレッドシート**を作成（または既存を流用し、**バックアップを取る**）。  
2. **拡張機能 → Apps スクリプト**で、このリポジトリの `.gs` と `WebApp.html` をコピーして貼り付け。  
3. スプレッドシートに戻り、メニュー **「育成出欠V1」→「シートを初期化」** を実行。  
4. **「設定」**シートで `予約ページURL` と `アラート送信先メール` を確認・修正。  
5. **「生徒マスタ」**に、既存4月表を見ながら生徒を登録（または **既存表取込_作業** から取込メニューを利用）。  
6. **生徒の閲覧トークン**：マスタで対象行を選び **「生徒の閲覧トークンを再生成」**（初回は空なら一括で可）。  
7. **Googleフォーム**を作成し、回答先スプレッドシートを **「フォーム回答」** シートに接続（または回答先をこのブックに指定）。  
8. フォーム1行目の見出しが、**設定シートの「フォーム_実施日列名」「フォーム_所属曜日列名」**と一致するよう調整（初期値は `実施日` / `所属曜日`）。  
9. **トリガー**：メニューから **フォーム送信トリガー登録**、**毎朝7時トリガー登録**。  
10. **デプロイ**：Apps スクリプトで **デプロイ → 新しいデプロイ → 種類：ウェブアプリ**、実行ユーザー：**自分**、アクセス：**全員**（閲覧者にトークンを渡す運用のため）。

## WebアプリURLの使い方

デプロイ後に表示される **ウェブアプリの URL** に、クエリを付けます。

```text
（ウェブアプリURL）?token=（生徒マスタの viewToken）
```

例：`https://script.google.com/macros/s/xxxx/exec?token=abc...`

---

## 時間主導トリガー

メニュー **「時間トリガーを登録（毎朝7時・未入力アラート）」** を実行すると、`Asia/Tokyo` の **7時** に `dailyAttendanceAlert_` が動きます。

- **前日**が月・火・木・金のいずれでもない場合は何もしません。  
- **前日**に対応する **実施日＋所属曜日** の行がフォーム回答に無い場合、メールを送ります。  
- 学校都合で実施が無い日は、**設定シートの「未入力アラート除外日」** に `yyyy-MM-dd` をカンマ区切りで入れてください。

## よくあるエラーと対処

| 現象 | 原因の例 | 対処 |
|------|-----------|------|
| フォーム送信でエラー | 実施日/所属曜日の列見出しが設定と不一致 | 設定シートの列名キーをフォームの1行目に合わせる |
| 出欠ログに誰も出ない | 生徒設問の見出しが氏名と不一致 | フォームの設問タイトルをマスタの氏名と完全一致 |
| Webアプリで開けない | デプロイ未実施・権限が「自分のみ」 | デプロイし直し、アクセスを「全員」に |
| 集計が古い | フォーム外でログを編集した | メニュー「集計を再計算」 |
| アラートが多い | 休校なのに未入力のまま | 除外日を設定する／前日にフォーム送信する |
| メールが送れない | クォータ・スパム判定 | 送信元アカウントの確認、宛先の受信設定 |
