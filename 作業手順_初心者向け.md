# 育成クラス出欠システム V1｜初心者向けセットアップ手順

このファイルは、**はじめて Google スプレッドシートや Apps Script に触れる方向け**に、画面で何をすればよいかだけを順番に書いたものです。  
**上から順に**進めてください。飛ばさないのがコツです。

---

## はじめに読む（2分）

### 何をする作業か

1. **スプレッドシート**（表のファイル）を用意する  
2. その中に **プログラム（Apps Script）** を入れる  
3. メニューから **表の準備** を自動で行う  
4. **生徒名簿** を入力する  
5. **Googleフォーム**（入力用アンケート）を作り、表につなぐ  
6. **保護者が見るページ（ウェブアプリ）** を公開する  
7. **毎朝のメール通知** をセットする  

### 用意するもの

- **パソコン**と**インターネット**（スマホだけだと大変なのでPC推奨）
- **Googleアカウント**（普段の Gmail でOK）
- このフォルダにある **`.gs` ファイル**と **`WebApp.html`**（コードをコピーします）
- **所要時間の目安**：初めての方で **60〜90分**（途中で休憩して大丈夫です）

### 用語（こんなイメージで大丈夫）

| 言葉 | イメージ |
|------|----------|
| スプレッドシート | Excel のような Google の表 |
| Apps Script | 表の裏側で動く小さなプログラム |
| トリガー | 「フォームが送られたとき」「毎朝7時」など、決まったタイミングで動かす設定 |
| デプロイ | プログラムを「ウェブのページとして公開する」作業 |
| トークン | 推測されにくい英数字の文字列。URL に付けて本人だけ見られるようにする |

---

## ステップ1｜新しいスプレッドシートを作る

1. ブラウザで [Google スプレッドシート](https://sheets.google.com) を開く  
2. **空白**を選んで新規作成  
3. 左上の名前をクリックして、わかりやすい名前に変える（例：`育成クラス出欠_V1`）  
4. **ファイル → コピーを作成** でバックアップを取っておいてもよい（失敗しても安心）

**ここまでできたら次へ。**

---

## ステップ2｜Apps Script を開く

1. スプレッドシートのメニューで **拡張機能 → Apps スクリプト** をクリック  
2. 別タブに **コード編集画面** が開きます（初めてなら真っ白な `コード.gs` が1つある状態）

**このタブは閉じずに、次のステップでコードを入れます。**

---

## ステップ3｜コードファイルを入れる（コピー＆貼り付け）

このプロジェクトには、次のファイルがあります。**それぞれ Apps Script 側に「ファイル」として作ります。**

### 3-1. いったん初期の `コード.gs` を消してよいか

- 最初にある **`コード.gs`** の中身を**すべて削除**しても、**ファイルごと削除**しても構いません（後で新しいファイルを足します）。

### 3-2. ファイルの作り方

1. Apps Script 画面左上の **「+」** または **「ファイルを追加」** から **スクリプト** を選ぶ  
2. ファイル名を **`Config`** にして保存（拡張子は自動で `.gs` になります）  
3. パソコンのフォルダから **`Config.gs`** を開き、**中身をすべてコピー**して、Apps Script の **`Config.gs`** に**貼り付け**  
4. **保存**（ディスクのアイコン、または Ctrl+S）

### 3-3. 同じように追加するファイル

次の **それぞれ**について、**新しいスクリプトファイルを追加**し、対応する `.gs` の中身を貼り付けて保存します。

| Apps Script 上のファイル名（例） | コピー元（このフォルダ内） |
|----------------------------------|----------------------------|
| `Config` | `Config.gs` |
| `Utils` | `Utils.gs` |
| `Setup` | `Setup.gs` |
| `Master` | `Master.gs` |
| `FormHandler` | `FormHandler.gs` |
| `Aggregation` | `Aggregation.gs` |
| `WebApp` | `WebApp.gs` |
| `Alerts` | `Alerts.gs` |
| `LegacyImport` | `LegacyImport.gs` |

**ポイント**

- ファイル名は **`Config.gs` ではなく `Config`** でOK（エディタが `.gs` を付けます）  
- **`doGet` は `WebApp.gs` にあります**（消さないでください）

### 3-4. HTML ファイルを追加する

1. **ファイルを追加 → HTML** を選ぶ  
2. 名前を **`WebApp`** にする（**`WebApp.html` ではなく `WebApp` でOK**。中身は HTML）  
3. フォルダの **`WebApp.html`** の中身をすべてコピーして貼り付け、保存  

### 3-5. `appsscript.json` について

- 画面左の **歯車「プロジェクトの設定」** を開く  
- **タイムゾーン** が **`Asia/Tokyo`** になっているか確認（違っていたら直す）  
- ローカルの `appsscript.json` の内容を上級者向けに合わせる必要は、**まずはなくて大丈夫**です  

**すべて保存できたら次へ。**

---

## ステップ4｜スプレッドシートに戻ってメニューを出す

1. **スプレッドシートのタブ**に戻る  
2. ページを **再読み込み（F5）** してもよい  
3. 数秒待つと、メニュー欄に **「育成出欠V1」** が出ます  

**出ないとき**

- Apps Script で **保存**したか確認  
- スプレッドシートを一度閉じて開き直す  
- 別のブラウザで試す  

---

## ステップ5｜初めての権限（承認）

1. メニュー **育成出欠V1 → シートを初期化（空シート作成・ヘルパー）** をクリック  
2. **権限の確認** と出たら **続行**  
3. Googleアカウントを選ぶ  
4. **詳細 →（安全ではないページ）に移動 → 許可** のような表示が出ることがあります  
   - 自分で作ったスクリプトなので、**許可**して進めます（初回だけの作業です）  
5. 終わると、いくつかのシート（**設定**・**生徒マスタ** など）ができます  

**エラーが出たら**、メッセージをメモして、後半の「困ったとき」を見てください。

---

## ステップ6｜「設定」シートを埋める

**「設定」** シートを開きます。

| キー | あなたがやること |
|------|------------------|
| **予約ページURL** | 振替予約に飛ばしたい **ホームページのURL** を貼る（まだなら仮で `https://example.com` でも可。後で直せます） |
| **アラート送信先メール** | 未入力通知を受け取る **あなたのメール**（例：`vnika0305@gmail.com`） |
| **未入力アラート除外日** | 最初は **空**でOK（休校日だけ後から `yyyy-MM-dd` で入れる） |
| **フォーム_実施日列名** | 最初は **`実施日`** のまま（フォームの列名と合わせます） |
| **フォーム_所属曜日列名** | 最初は **`所属曜日`** のまま |

**保存は自動です**（スプレッドシートは編集するとそのまま保存されます）。

---

## ステップ7｜生徒マスタを入力する

**「生徒マスタ」** を開きます。

1. **サンプル行**（例：サンプル太郎）があれば、**削除してOK**です  
2. **1人につき1行**で入れます  

| 列 | 意味 | 例 |
|----|------|-----|
| studentId | 識別番号（被らなければOK） | `S001` |
| 氏名 | **フォームの設問名と同じ字**にする | `山田太郎` |
| コース | 週1 / 週2 / 週3 | `週2` |
| レギュラー曜日 | カンマ区切り | `月,木` |
| 初期残振替 | 今の残回数が分かれば数字。分からなければ `0` | `0` |
| viewToken | まだ空でOK（次のステップで入る） | 空 |
| token無効 | 空でOK | 空 |
| メモ | 任意 | 空でOK |

**コツ**：後で作る **Googleフォームの「生徒ごとの設問のタイトル」** を、**この「氏名」と完全に同じ**にします。スペースや「さん」まで揃えると安全です。

---

## ステップ8｜閲覧用トークンを発行する

1. **生徒マスタ**で、トークンを付けたい行を **ドラッグして選択**（複数行OK）  
2. メニュー **育成出欠V1 → 生徒の閲覧トークンを再生成（選択行のみ）**  
3. **viewToken** 列に長い文字列が入ればOK  

**全員まとめて**やりたいときは、データがある行をすべて選んで同じメニューを実行します。

---

## ステップ9｜Googleフォームを作る

1. [Googleフォーム](https://forms.google.com) で **空白のフォーム**  
2. タイトルを付ける（例：`育成クラス 出欠入力`）  

### 必須の質問（最初の方）

1. **実施日** … 種類：**日付**  
2. **所属曜日** … 種類：**ラジオボタン** または **ドロップダウン**。選択肢は **`月` `火` `木` `金`** だけ（ひとつ選ぶ）  
3. （任意）**コーチ名** … 短文  

#### 生徒ごとの質問（ここが一番大事）

- **セクション**を使って、「**所属曜日＝月** のときだけ月曜の生徒のブロックを出す」ようにするとミスが減ります（フォームの **歯車 → セクションに分割** などで設定）。  
- **各生徒について1問**つくり、種類は **複数選択肢（ラジオ）** で **`出席` `欠席` `振替出席`** の3つ。  
- **質問のタイトル**を、**生徒マスタの「氏名」と一字一句同じ**にする（例：`山田太郎`）。

**コツ**

- 最初は **1クラス分だけ**（例：月曜の数名）で試し、動いたら増やすと安心です。

---

## ステップ10｜フォームの回答先をこのスプレッドシートにする

1. フォームの **回答** タブを開く  
2. **スプレッドシートにリンク** を選ぶ  
3. **既存のスプレッドシートを選択** し、先ほど作った **`育成クラス出欠_V1`** を指定  
4. できあがった回答用シートの名前が **`フォーム回答`** になっているか確認  

**名前が違う場合**

- シート名を **`フォーム回答`** に**リネーム**する（コードがその名前を探します）  
  または、コード側のシート名を変える（上級者向け）ので、**まずは名前を合わせる**のが簡単です。

### 1行目（見出し）の確認

- フォームを1回 **テスト送信** します  
- **`フォーム回答`** の **1行目** に、`実施日` と `所属曜日` という文字が並んでいるか見る  
- **設定**シートの **フォーム_実施日列名**・**フォーム_所属曜日列名** と **同じ表記**になっているか確認  

違っていたら、**設定の値を1行目に合わせる**か、**フォーム側の質問タイトルを変える**か、どちらかで揃えます。

---

## ステップ11｜トリガーを登録する（2種類）

スプレッドシートに戻り、メニューから順に実行します。

1. **育成出欠V1 → フォーム送信トリガーを登録**  
2. **育成出欠V1 → 時間トリガーを登録（毎朝7時・未入力アラート）**  

**確認（任意）**

- Apps Script 画面左の **時計アイコン（トリガー）** を開き、一覧に表示されていればOKです。

---

## ステップ12｜Surge で CSS を公開する（見た目用）

保護者向けページの **HTML/CSS は Surge の公開URL** から読み込みます（**このステップを先に**済ませてから、次の「ウェブアプリ」をデプロイするとスムーズです）。

1. PCに **Node.js** が入っているか確認（なければ [nodejs.org](https://nodejs.org/) からインストール）  
2. **PowerShell** などで、このプロジェクトの **`surge`** フォルダへ移動する  
3. 次を実行する（`my-ikusei-v1` の部分は、**英数字とハイフン**で好きな名前。世界で重複しない名前にする）  

```text
npx surge . my-ikusei-v1.surge.sh
```

4. 初回は Surge の **メール・パスワード** を聞かれるので、画面の指示でアカウント作成またはログイン  
5. 完了すると **`https://my-ikusei-v1.surge.sh/styles.css`** のようなURLが表示される（表示されたURLをメモ）  
6. スプレッドシートの **「設定」** シートで **「Surge_スタイルシートURL」** の欄に、**`.../styles.css` まで** を貼り付ける  
   - シートにその行がまだ無い場合は、**キー** に `Surge_スタイルシートURL`、**値** にURLを追加  
7. （任意）**`Config.gs`** の **`SURGE_STYLESHEET_URL`** も同じURLにそろえる  

**プレビュー**：ブラウザで `https://（同じサブドメイン）.surge.sh/index.html` を開くと、デザインだけ確認できます（本番の生徒データは入りません）。

詳細は **`surge/README.md`** も参照してください。

---

## ステップ13｜ウェブアプリ（保護者ページ）を公開する

※ **ステップ12（Surge）** で CSS のURLを **設定** に入れておいてください。

1. **Apps Script** のタブを開く  
2. 右上 **デプロイ → 新しいデプロイ**  
3. **種類の選択** で **ウェブアプリ**  
4. 次のように設定  
   - **次のユーザーとして実行：自分**  
   - **アクセスできるユーザー：全員**（トークンを知っている人だけが見られる運用のため）  
5. **デプロイ** をクリック  
6. **ウェブアプリのURL** が表示されるので **コピー**してメモ  

### 保護者に渡すURLの形

```text
（コピーしたウェブアプリのURL）?token=（生徒マスタの viewToken）
```

例：`https://script.google.com/macros/s/xxxx/exec?token=abcd1234...`

**1人につき1つのURL**になります（トークンが違う）。

## ステップ14｜動作テスト（必ず）

1. **フォーム**からテスト送信（実施日は今日、所属曜日はそのクラスに合わせる）  
2. **`出欠ログ`** に行が増えているか  
3. メニュー **育成出欠V1 → 集計を再計算（全員）** を実行（念のため）  
4. **`集計`** シートに数字が出ているか  
5. ブラウザで **`ウェブアプリURL?token=...`** を開き、名前と残数が見えるか

## 困ったとき（よくあるつまずき）

| 症状 | 確認すること |
|------|----------------|
| メニュー「育成出欠V1」が出ない | Apps Script を保存したか、表を再読み込みしたか |
| フォーム送信後にエラー | **氏名**がフォームの設問名と**完全一致**か。**実施日・所属曜日**の列名が**設定と一致**か |
| 出欠ログが空 | その曜日の生徒として、フォームに**質問が出ている**か。回答が空ではないか |
| ウェブアプリが開けない | **デプロイ**したか。**アクセス：全員**か。URL に `?token=` を付けたか |
| 画面のデザインが崩れる | **Surge** の `styles.css` URLが **設定** に正しく入っているか。**https** で開けるか |
| メールが来ない | **トリガー**が登録されているか。前日が月・火・木・金だったか。**除外日**に入っていないか |
